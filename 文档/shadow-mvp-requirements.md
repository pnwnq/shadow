# "Shadow" 实验室管理系统 - MVP 需求文档

## 1. 引言

### 1.1 项目背景
当前实验室面临资料查找困难、物品管理混乱、项目进度不透明等痛点。"Shadow" 项目旨在创建一个核心的实验室管理系统，解决这些关键问题，提升基础运营效率。该文档定义了项目的最小可行产品（MVP）范围。

### 1.2 MVP 目标
*   **核心目标:** 交付一个稳定、可用的基础管理平台，实现人员、物品、基础文档和项目的初步数字化管理。
*   **具体目标:**
    *   实现安全的身份认证和基础的角色权限控制。
    *   提供可靠的物品借还追踪功能，减少物品丢失。
    *   建立集中的文档存储库，方便查找和共享基础资料。
    *   提供简单的项目和任务跟踪能力。
    *   搭建系统核心技术框架，为后续迭代打下基础。

## 2. 用户角色与权限
系统需支持以下核心用户角色，并实现基础的基于角色的访问控制 (RBAC)：

*   **管理员/老师:** 负责用户管理、角色分配、物品分类管理、系统基础配置。
*   **普通成员:** 实验室主要使用者，可借还物品、上传/查阅文档、参与项目。
*   *(简化: 移除非核心的开发者、项目负责人、访客角色，权限合并或后续添加)*

## 3. 功能性需求 (MVP 范围)

### 3.1 用户管理系统 (核心基础)
*   用户注册、登录、登出。
*   用户信息基础管理（管理员操作）。
*   角色分配（管理员操作）。
*   基于角色的页面/基础功能访问控制。

### 3.2 物品管理系统 (核心痛点)
*   **物品信息管理:** 物品录入、分类、标签管理（管理员配置，成员查看）。
*   **库存管理:** 实时显示物品基本状态（如：在库、借出）。
*   **借还流程:**
    *   **核心:** 手动选择物品进行借用登记和归还登记。
    *   **记录追踪:** 记录物品借还时间、借用人。
    *   *(简化/推迟: 初期不强制要求NFC集成，可用手动或二维码/条形码替代，NFC作为后续增强功能)*
*   **列表与查询:** 提供物品列表，支持按名称、分类、状态等基础条件查询。

### 3.3 基础文档管理系统 (核心痛点)
*   **上传与存储:** 支持常见文档格式（PDF, Word, 图片等）上传，集成文件存储（如MinIO或本地存储）。
*   **列表与查看:** 提供文档列表视图，支持按名称、上传者、分类等排序和筛选。
*   **基础信息:** 查看文档基本信息（名称、大小、上传时间、上传者）。
*   **在线预览:** *(强烈建议保留)* 支持常见格式（如PDF, 图片）的在线预览。
*   **基础搜索:** 提供基于文档标题或文件名等的数据库 `LIKE` 搜索。
*   **分类与标签:** 支持管理员创建和管理文档分类和标签，成员可按分类/标签浏览。
*   **权限控制:** 实现基于角色的基础文档访问权限（如：成员可查看，管理员可管理）。
*   *(简化/推迟: 移除在线编辑、高级共享、版本控制、全文搜索、统计分析)*

### 3.4 基础项目管理系统 (核心痛点)
*   **项目创建与管理:** 管理员或指定角色可创建项目，录入基本信息（名称、描述）。
*   **成员管理:** 将用户分配到项目中。
*   **任务管理:**
    *   项目内创建任务，分配给成员。
    *   任务包含基本信息（标题、描述、负责人、截止日期-可选）。
    *   任务状态跟踪（如：待办、进行中、已完成）。
*   **列表与查看:** 提供项目列表和项目内的任务列表。
*   *(简化: 聚焦核心的任务分配和状态跟踪，移除复杂的流程、甘特图等)*

### 3.5 基础学习路径/资源 (核心痛点 - 简化版)
*   **资源列表:** 提供一个集中的页面，由管理员手动录入和维护学习资源列表（如：文档链接、外部教程链接、书籍推荐）。
*   **分类展示:** 可以对资源进行简单的分类（如：入门、单片机、PCB设计等）。
*   *(简化/推迟: 移除AI个性化推荐和复杂的学习路径规划)*

## 4. 非功能性需求 (MVP 核心)

*   **可用性:**
    *   系统稳定可靠，核心功能（登录、物品借还、文档上传下载）运行正常。
    *   界面简洁、直观，操作流程清晰
*   **性能:**
    *   常规操作（页面加载、列表查询、物品登记、文档上传下载）应有可接受的响应速度。
*   **安全性:**
    *   实现安全的密码存储和身份验证。
    *   基础的角色权限控制有效。
*   **可维护性:**
    *   代码结构清晰，遵循基本开发规范。
*   **兼容性:**
    *   前端兼容主流现代浏览器（Chrome, Firefox, Edge）。


## 6. 未来考虑 (MVP 后迭代)
以下功能被移出MVP范围，可在核心功能稳定运行后，根据优先级考虑加入：
*   AI智能助手（问答、个性化推荐）
*   NFC集成物品管理
*   文档在线编辑、高级共享、版本控制、全文搜索(ES)、统计分析
*   更复杂的项目管理功能（甘特图、流程自定义等）
*   竞赛管理、签到、通知系统等附加功能
*   用户角色细化（开发者、项目负责人、访客等）
